# Noridoc: nori-integrations-toolshed

Path: @/skills/nori-integrations-toolshed

### Overview
- Contains the shared agent skill that documents all CLI tools available in the nori-integrations toolshed
- [`SKILL.md`](SKILL.md) is auto-symlinked into `~/.claude/skills/toolshed/` on sprites by the broker infrastructure, making it discoverable by coding agents

### How it fits into the larger codebase
- This directory lives under [@/skills](../), which is the toolshed skills directory -- distinct from `.claude/skills/` which contains skills for developing on this repo
- The broker symlinks the entire `skills/` tree into `~/.claude/skills/toolshed/` when provisioning sprites, so agents automatically see this skill
- `~/AGENTS.md` (generated by [@/setup.sh](../../setup.sh)) references this skill by name, directing agents here for detailed usage of each CLI tool

### Core Implementation
- [`SKILL.md`](SKILL.md) documents each toolshed-provided CLI tool (nori-slack, gws, sprite, gam) with usage examples, required environment variables, and availability-check commands
- The skill acts as the single source of truth for how to use toolshed CLIs on a sprite

### Things to Know
- `~/AGENTS.md` provides a brief list of available CLIs; this skill provides the detailed usage reference
- Tools that failed setup are omitted from `~/AGENTS.md` but are still documented in this skill -- agents should use `command -v` to check availability before use
- The `nori-broker` CLI is intentionally NOT documented here -- it is delivered separately through the broker server's base bootstrap bundle (canonical source: `sessions/broker/cli/` in the `tilework-tech/sessions` repo) and is available on every sprite regardless of which toolshed an org configures
